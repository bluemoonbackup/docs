Upgrade Notes ¶

Upgrading from Blue Moon Backup Gen 1 ¶

These upgrade notes refer to upgrading the backup client from first generation (v6) to second generation (v20). Blue Moon Backup through version 6.x is referred to as our first generation. No versions between 7 and 19 exist. Version 20.x onward is considered 2nd generation. Upgrades after version 20 are pushed automatically from the server. Typically no action is required on your part after version 20.

Note: the v6 client is referred to as "existing" or Blue Moon Backup. The v20+ client is referred to as "Cloud Backup".

Cloud Backup can import settings from an installed version of Blue Moon Backup 6.x, subject to the following notices and conditions:

  • Backupsets are imported as Protected Items.
    • Backupset types
      • File
        • Source selection/deselection is supported.
        • Macro selections (desktop, documents etc) are not supported.
        • Inclusion/exclusion filters are not supported.
      • MySQL
      • Microsoft SQL Server
        • The "All instances" option is not supported. Your existing backupset must have selected specific instances for backup.
      • Other backupset types are not imported and must be created manually.
    • Pre/post commands
      • Pre/post commands are imported as applying to the Protected Item, not to the Storage Vault nor the Schedule.
    • Retention policies
      • Basic retention policies (DAYS/JOBS -type) are supported.
      • Advanced retention policies will be treated as "keep forever".
    • Schedule
      • Daily, Weekly and Custom schedules are supported.
      • Monthly schedules for a specific day of the month are supported.
      • Monthly schedules for a variant day of the month (e.g. Second Tuesday, Last Weekday) are not supported.
    • "Extra Local Copy"
      • "Local Copy" will be imported as a new or existing Local Path Storage Vault in Cloud Backup. You may want to choose an alternate location and may want to delete the existing local copy to free up disk space since Cloud Backup will create a new (duplicate) copy.
      • The "Skip Offsite Backup" option is correctly imported.

The new retention policy is by default 365 days. Existing backup sets will be imported with their old retention settings. Consider changing the retention policy back to default in the retention tab of each job. Your backup rep can do this centrally, or it can be done from the web client. Note: when viewing settings in the Cloud Backup client the protected items list is by default filtered to items that belong to the local machine.

Please verify that each protected item completes successfully.
